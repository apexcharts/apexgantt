ApexGantt — UI State Persistence

Zoom, scroll, expand/collapse, selection, sort, and filter are captured as a serializable snapshot via getState() / setState(). This demo also enables persistState, so the view is remembered across reloads via localStorage.

How to interact
  • Change the view: sort a column, collapse a phase, select a row, or zoom (Ctrl + scroll).
  • This chart has persistState: true — reload the page and the view comes back exactly as you left it.
  • Save snapshot captures the current state; mess it up then Restore snapshot to see setState() put it back.
  • Reset clears the persisted state from localStorage.
